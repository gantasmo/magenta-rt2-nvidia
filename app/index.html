Quickstart

MRT2 Studio generates an audio track from a text description using the mrt2_small model on the local GPU.

  1. On the Compose tab, enter a description and tap genre tags to stack weighted vibes. Click a tag's bars to quantize how strongly it counts.
  2. On the Sound tab, set length and other parameters with the vertical faders. The defaults suit most prompts.
  3. The Mix & Notes tab blends extra style sources or audio, steers the melody by pitch, and locks a style seed.
  4. Press Generate. The first run after launch compiles the model and takes 30 to 60 seconds. Later runs are faster.
  5. The track plays in the waveform strip at the top and is written to app\output\ as a WAV file. It also appears in the Library on the right.
  6. Press Extend to add audio to the current track. A changed prompt before Extend morphs the continuation into the new description without a cut.

API reference

The Studio server runs inside WSL2 and listens on http://localhost:8777. The active port is recorded in .studio_port and steps higher when the default is busy.

GET /health

Returns engine status as JSON.

{
  "app": "mrt2-studio",
  "ready": true,
  "status": "ready",
  "error": null,
  "model": "mrt2_small",
  "device": "cuda:0"
}

POST /generate

Accepts a JSON body and returns audio/wav bytes. Unset fields fall back to defaults.

FieldTypeDefault
promptstring"warm analog pads"
durationseconds10
temperaturenumber1.3
top_kinteger40
cfg_musiccocanumber3.0
cfg_notesnumber1.0
cfg_drumsnumber1.0
drums-1, 1, 0-1
styleslist of {type, text/audio_b64, weight}none (uses prompt)
notesMIDI numbers or 128-vectornone
seedinteger0

Response headers report timing and output details:

HeaderMeaning
X-Generate-SecondsCompute time for the segment
X-Audio-SecondsLength of the full track
X-Segment-SecondsLength of the new segment
X-RTFReal-time factor, segment seconds divided by compute seconds
X-Sample-RateOutput sample rate in Hz
X-FilenameSaved file name in the output folder

POST /extend

Takes the same body as /generate. Continues the current piece from the model state and returns the full track. A changed prompt morphs the continuation.

POST /shutdown

Stops the server process. The Quit button calls this endpoint.

Example

curl -s "http://localhost:8777/generate" -H "Content-Type: application/json" -d "{\"prompt\":\"dreamy lofi piano\",\"duration\":8}" --output track.wav

Troubleshooting

SymptomCause and resolution
Status reads "server offline" or the spinner does not clearThe engine process stopped. Re-open MRT2-Studio, or run Setup-MRT2.bat to repair.
The overlay shows "Engine error"The model failed to load. Setup-MRT2.bat re-downloads and verifies the model.
The first generation is slowThe model compiles once per launch, which takes 30 to 60 seconds.
The player produces no soundThe browser requires one click on the player before audio starts.
A generation stops partwayThe browser closed the connection. Reload the page and generate again.
WSL warns about processors or memorySetup-MRT2.bat writes a .wslconfig sized to the machine. Run it again to apply.
A page titled "starting" opens instead of the StudioThe engine did not report a port. The page forwards to the Studio once the engine answers, and otherwise links back to Setup-MRT2.bat.

About

MRT2 Studio is an NVIDIA/CUDA port of Magenta RealTime 2, built by GANTASMO as part of theDAW. It was created at the Music Hackspace Music Technology Hackathon at Berklee College of Music.
